# 🔥 6. LangChain4j Integration (TRICKY PART)

LangChain4j currently is **text-first**, but supports:

```java
UserMessage.from(
    TextContent.from("Describe image"),
    ImageContent.from(url)
)
```

---

## Adapter Mapping

```java
public MultiModalMessage map(UserMessage msg) {

    List<ContentBlock> blocks = new ArrayList<>();

    for (Content content : msg.contents()) {
        if (content instanceof TextContent t) {
            blocks.add(new TextBlock(t.text()));
        }
        if (content instanceof ImageContent i) {
            blocks.add(new ImageBlock(i.url()));
        }
    }

    return new MultiModalMessage(USER, blocks);
}
```

# 🔥 3. Two Types of Fusion (You MUST support both)

---

## 🟢 Early Fusion (token-level)

Used by:

* LLaVA
* Qwen-VL

```id="q0l2rm"
[text tokens] + [image tokens] → concatenated sequence
```

👉 Pros:

* simple
* fast

👉 Cons:

* limited reasoning

---

## 🔵 Late Fusion (cross-attention)

Used by:

* Flamingo
* Gemini-like systems

```id="1p7p0s"
text attends to image embeddings dynamically
```

👉 Pros:

* powerful reasoning

👉 Cons:

* complex
* kernel-dependent
